# 06 — Checklists

## A. Sales / discovery checklist

- confirm lead volume and lead sources
- confirm buyer / seller / investor focus
- identify single biggest leakage point
- define qualified conversation
- confirm current CRM and messaging stack
- confirm who owns hot lead follow-up
- confirm access can be granted within 48 hours
- narrow scope to one 7-day wedge
- state out-of-scope items explicitly

## B. Pre-kickoff checklist

- agreement approved
- payment confirmed if required
- client owner assigned
- kickoff scheduled
- access request sheet sent
- sample lead data requested
- success metrics documented
- reporting recipients confirmed

## C. Build checklist

- lead sources mapped
- required fields mapped
- lifecycle stages mapped
- routing rules documented
- qualification prompts / scripts drafted
- nurture sequence drafted
- reactivation sequence drafted if in scope
- alerting path configured
- reporting view configured
- fallback manual process documented

## D. QA checklist

Run at least 5 sample records through each in-scope path.

- new lead creates correctly
- source attribution preserved
- owner assignment works
- auto-response triggers correctly
- hot lead escalation works
- nurture enrollment works
- opt-out handling works
- invalid / duplicate handling works
- CRM stage updates correctly
- report totals match sample activity

## E. Launch checklist

- client sign-off received
- business hours set correctly
- alert recipients confirmed
- final templates approved
- test leads removed or tagged
- live lead monitoring owner assigned
- first daily report scheduled
- escalation contact documented

## F. Day-1 post-launch checklist

- verify first live leads flowed in
- verify no routing failures
- verify first response SLA met
- verify hot alerts reached the right human
- verify tasks / next actions were created
- verify report generated cleanly
- log all defects and tuning items

## G. CRM hygiene checklist

Daily:

- review duplicates
- review unassigned leads
- review active leads with no next action
- review leads stuck over SLA
- review invalid contact queue

Weekly:

- audit stage consistency
- audit owner compliance
- audit response times by source
- audit nurture / reactivation performance
- archive dead test data

## H. Client handoff checklist

At end of pilot, client should receive:

- workflow map
- routing rules summary
- stage definitions
- sequence library
- daily report spec
- hot lead SOP
- admin notes for future edits

## I. Expansion checklist

Only expand after week-one system is stable.

- wedge 1 performing for at least 7-14 days
- reporting trustworthy
- owner SLA being followed
- one new source or persona selected
- scope and timeline re-approved
